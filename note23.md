## 스택(Stack)
데이터를 저장할 수 있고, 최근에 삽입한 데이터부터 출력되는 자료구조

### 후입 선출(LIFO, Last-In_First-Out)
삽입했던 데이터를 출력할 때, 가장 최근에 들어론 데이터가 가장 먼저 나가는 특징

푸쉬(Push)와 팝(Pop)
푸쉬(Push):스택에 데이터를 삽입하는 작업
팝(Pop): 스택에서 데이터를 출력 및 삭제하는 작업

### 스택의 동작 방식
스택은 깊은 바구니 같은 모양으로 표현
데이터를 집어넣으면 먼저 넣은 데이터부터 아래에서부터 천천히 쌓임
출력시 최근에 넣은 데이터부터 출력

### 자바에서 스택 사용
스택(Stack) 클래스
    자바에서 기본으로 제공하는 클래스
    삽입을 위해 push(E e) 메소드 사용
    조회 / 삭제를 위해 pop() 메소드 사용
    조회를 위해 peek() 메소드 사용

### 큐 vs 스택

    큐(Queue)
입츌력 순서 : 선입선출(FIFO)
삽입연산의 이름 : enqueue
출력(삭제) 연산의 이름: dequeue
사용 예시:
    대기열 순서
    작업 예약
    너비 우선 탐색(BFS, Breadth-First-Search) 구현
e.g.: 터널, 먼저 진입 한 차가 먼저 나감

    스택(Stack)
입출력 순서: 후입선출(LIFO)
삽입연산의 이름: push
출력(삭제) 연산의 이름:pop
사용 예시:
    웹 브라우저 뒤로가기
    실행 취소(Undo)
    깊이 우선 탐색(DFS, Depth-First Search) 구현
    수식의 괄호와 연산 순서 검사
e.g.: 엘리베이터, 마지막에 탄 사람이 먼저 내림

### 실생활 예시
문서나 이미지 작업의 실수를 되돌려주는 되돌리기(Undo) 기능
대부분의 문서 / 이미지 작업 프로그램은 스택에 작업 이력을 저장하고, 최근 작업 내역을 제거해서 이전 상태로 되돌아갈 수 있음

### IT 개발 현장 예시
콜 스택(Call Stack)
    프로그램이 실행중에 호출한 함수의 목록을 관리하는 자료구조
    새 함수가 호출되고, 실행이 끝나면 이전 함수로 되돌하가는 동작에 스택을 사용
    프로그램의 결함을 고치는 디버깅 과정에서 콜스택을 활용
