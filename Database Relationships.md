## Database Relationships(데이터베이스 관계)

### 1:N 관계
1:N 관계(One to Many Relationships)은 한 테이블의 한 행이 다른 테이블의 여러 행과 관련이 있는 경우를 의미

### M:N 관계
M:N 관계(many to many relationship)는 한 테이블(A)의 한 행이 다른 테이블(B)의 여러 행과 관련이 있고, 반대로 한 테이블(B)의 한 행이 다른 테이블(A)의 여러 행과 관련이 있는 경우를 의미

### 연관 테이블(중계 테이블)
연관 테이블은 M:N 관계에서 두 테이블 간의 관계를 표현하기 위해 사용
    M:N 관계는 관계형 데이터베이스에서 직접 표현할 수 없어, 연관 테이블을 통해 두 개의 1:N 관계로 분해하여 표현
연관 테이블에는 두 테이블의 기본 키(PK)가 외래 키(FK)로서 포함

### 1:1 관계(one to one relationship)
1:1 관계(one to one relationship)는 한 테이블의 한 행이 다른 테이블의 단 하나의 행과만 관련이 있는 경우를 의미
1:1 관계는 주로 다음과 같은 상황에서 사용
    보안이 필요한 민감한 데이터 분리
    자주 사용하는 데이터와 그렇지 않은 데이터 분리
    선택적으로 존재하는 데이터 관리
    매우 큰 데이터를 별도로 관리
