### 자료구조 기초

연결 리스트(Linked List):
    여러 노드의 연결로 이루어진 리스트
    각 노드는 데이터 필드와 다음 노드로의 참조 링크를 담고 있음
특징:
    동적인 길이: 각 노드가 메모리 상에 연속적으로 위치할 필요가 없으므로 길이의 제약이 적고, 배열 리스트와 달리 내부적으로 배열을 사용하지 않으므로 리스트의 길이 변화에 부담이 적음
    성능 효율: 각 리스트의 앞과 뒤에 데이터를 추가하거나 삭제하는 것이 효율적
리스트(List)의 대표적인 구현제

실생활 예시:
    다양한 용도
        연결리스트는 다양한 용도를 가진 범용적인 자료구조
    예시
        이미지 뷰어나 음악 플레이어의 목록 보관
        큰 숫자의 산술연산 또는 다항식의 표현
        작업 스케쥴링
        트리와 그래프 구현
        큐 또는 스택 구현

IT 개발 현장에서 쓰임새?
    데이터베이스 색인(인덱스, Index)
        빠른 조회를 위해 데이터베이스 테이블을 색인하고 색인 정보를 보관함
        더 효율적인 색인 검색, 삽입, 삭제를 위해 연결 리스트를 개선한 B+ 트리를 적용
    B+ 트리
        데이터는 최하단의 단말(Terminal) 노드에 있고, 각 단말 노드는 연결 리스트의 형태를 지님
        연결 리스트보다 리스트의 중간에 데이터를 삽입 또는 삭제하는 성능이 좋음

심화 주제
    큐(Queue)
    스택
    B+트리
    데이터베이스 색인 - 연결리스트를 B+트리가 사용하고 B+트리는 데이터베이스 색인에 사용됨
