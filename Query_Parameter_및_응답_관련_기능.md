### Query Parameter 활용
URL 뒤에 ?key=value 형태로 전달되는 쿼리 파라미터를 활용할 수 있음  
path parameter에 명시되지 않은 함수의 입력값이 query parameter로 활용  

### 클라이언트 -> 서버로의 데이터 전달 방식
- Path Parameters (경로 매개변수)  
URL 경로의 일부로 데이터를 전달  
주로 특정 리소스를 식별할 때(ID 등) 사용  

- Query Parameters (쿼리 매개변수)  
URL 뒤에 ?로 시작하여 key=value 형태로 전달  
주로 정렬, 필터링, 검색 등 선택적인 옵션을 부여할 때 사용  

- Request Body (요청 본문)  
데이터를 JSON 형태로 본문에 담아 전달  
주로 데이터를 생성(POST)하거나 수정(PUT)할 때, 복잡하고 양이 많은 데이터를 보낼 때 사용  

### Response Model (응답 모델)
- 서버가 클라이언트에게 데이터를 돌려줄 때 사용하는 필터  
- 동일한 데이터모델이라도 상황(로그인, 타인 프로필 조회 등)에 따라 보여줄 정보를 다르게 설정할 수 있으며, 비밀번호, 주민번호 같은 민감한 정보를 제외하고 내보낼 수 있음  
- DB 구조가 바뀌어도 클라이언트가 받는 데이터 형식은 유지할 수 있음  
- 요청 모델(PostCreate)과 더불어 DTO(Data Transfer Object)라고 불림  
- 각 응답에 맞는 Response Model을 사용할 수 있음  

### 상태 코드 설정
200 OK : 조회 성공  
201 Created : 생성 성공  
204 No Content : 삭제 성공(반환할 데이터 없음)  
404 Not Found : 데이터를 찾을 수 없음  

### 예외 처리
데이터가 없을 때 단순히 메세지를 보내는 것이 아니라, HTTPException을 사용하여 404 상태 코드를 명확히 전달  
일반적으로 예외가 raise될 경우 상위 어딘가에서는 반드시 try-except를 해주어야 하고, 그렇지 않으면 프로그램이 종료됨  
FastAPI에서는 프로그램이 종료되지 않고 적절한 응답으로 예외처리를 해주며, 이후에 자세한 에러 핸들링을 다룰 예정
