### 자료구조 기초

리스트(list):
    복수의 데이터를 사용자가 지정한 순서대로 저장하는 자료구조
특징:
    데이터의 목록을 표현하는 추상적인 개념이며, 다양한 구현 방식이 있음
    목록을 구성하는 데이터는 굳이 메모리에 련속적으로 위치라지 않아도 됨
    목록을 구성하는 데이터의 객체는 메모리 상 여러곳에 존재하며, 리스트는 각 데이터의 주소 또는 참조 정보만 지니고 있음
다양한 구현체
    리스트는 추상적인 개념이며, 여러 구체적인 구현 방식이 있음
    대표적인 구현체로 배열 리스트(Array List)와 연결 리스트(LinkedList)가 있음

배열 리스트(Array List)
    내부적으로 배열을 이용해 구현
    각 배열의 원소는 데이터 객체에 접근할 수 있는 참조 정보를 보관하고 있음
    인덱스를 통한 임의 접근이 빠름
    리스트 내부의 배열의 크기가 허용하는 한, 데이터 추가나 마지막 데이터 제거가 용이함
    리스트의 크기가 내부 배열을 넘는 경우, 더 큰 배열을 다시 생성하기 떄문에 성능이 떨어질 수 있음
    리스트 중간에 데이터를 삽입 또는 제거하는 경우, 내부 배열의 원소들을 밀거나 당겨야 하기 때문에 성능이 떨어질 수 있음

연결 리스트(Linked List)
    리스트의 각 원소는 데이터 객체의 팜조 정보와 다음 원소의 참조 정보를 함꺠 보관하고 있음
    임의 접근이 상대적으로 느리지만 리스트의 원소들을 순차적으로 순회하거나 삭제 / 추가할 때 성능이 좋음
    리스트 길이의 제약이 적음

배열 리스트의 선언과 초기화
    Arraylist 클래스의 객체로 선언
    Lost<Strinf> list - new ArrayList<>();

행렬에서 할 수 있는 것들
    인덱스로 데이터 접근 - System.out.print(list.get(0));

        list.add(" ");

    데이터 삭제 - list.remove(1);

실생활 사례
다양한 용도 - 리스트는 다양한 용도를 가진 범용적인 자료구조
예시:
    숫자 또는 문자열 목록 정렬
    할 일 목록
    기론, 풍속과 같은 연속적 데이터 보관
    쇼핑 카트이 목록 저장
    알고리즘이나 프로그램 수행 절차 보관
    큐 또는 스택 구현

IT 개발현장에서의 쓰임새?
데이터베이스(Database)의 테이블(Table)
    데이터베이스 테이블은 같은 구조를 가진 데이터를 선형으로 보관
    하나의 행(Row)은 여러 열(Column)을 포함하는 구조체
    행은 각 리스트의 원소에 대응함
    수백~수억의 대용량 자료를 효율적으로 보관하고 수정할 수 있어야 하기 때문에 다양한 기술을 사용해서 구현해야 함
