## 파이썬 가상환경과 uv
파이썬 가상환경은 프로젝트별로 독립적인 실행 환경을 구성하는 기술  
시스템 전체에 설치된 패키지와의 충돌을 방지하고, 프로젝트마다 서로 다른 버전의 패키지를 사용할 수 있도록 격리된 공간을 생성  

### uv의 정의 및 특징
uv는 러스트로 작성된 파이썬 패키지 및 프로젝트 관리 도구  
기존의 pip, pip-tools, virtualenv를 대체하며 처리 속도가 빠르고 단일 실행 파일로 동작  

### 프로젝트 초기화 및 가상환경 생성
uv를 사용하여 새로운 프로젝트를 생성하면 가상환경 설정과 의존성 관리 파일인 pyproject.tml이 자동으로 구성  

### 가상환경 활성화
생성된 가상환경을 현재 터미널 세션에 적용하기 위해서는 활성화 과정이 필요  
운영체제에 따라 활성화 스크립트의 경로가 다름  
가상환경을 활성화하면 terminal에 가상환경의 이름이 나타나게 됨  
deactivate를 통해 가상환경을 비활성화 할 수 있음  

### 패키지 설치 및 의존성 관리
uv는 프로젝트의 의존성을 관리하기 위해 uv.lock 파일을 사용  
이는 패키지의 정확한 버전을 기록하여 환경 간 동일성을 보장  

### 가상환경 내 스크립트 실행
가상환경을 수동으로 활성화하지 않고도 uv run 명령어를 사용하여 가상환경 내의 파이썬 인터프리터로 스크립트를 즉시 실행할 수 있음  

### 의존성 동기화
다른 환경에서 공유받은 pyproject.toml 파일을 바탕으로 현재 가상환경의 패키지 상태를 일치시킬 때 사용  

### pyproject.toml와 uv.lock의 차이
- pyproject.toml  
    사람이 직접 수정 가능하며, 필요한 패키지와 대략적인 버전 범위를 기록
- uv.lock  
    uv가 자동으로 관리하며, 실제로 설치된 패키지의 정확한 버전과 해시값을 기록
