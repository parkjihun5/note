## HTTP

### HTTP 프로토콜
HTTP(HyperText Transfer Protocol)는 웹에서 데이터를 주고받기 위한 통신 규약
클라이언트와 서버 사이에서 메세지를 어떻게 주고받을지 정해놓은 약속

### 클라이언트와 서버 구조
클라이언트: 서비스를 요청하는 주체. 웹 브라우저나 모바일 앱이 해당
서버: 요청에 대해 응답을 제공하는 주체

### HTTP의 주요 특징
비연결성
    클라이언트가 요청을 보내고 서버가 응답을 마치면 즉시 연결을 끊는 방식
    이를 통해 서버의 자원을 효율적으로 관리할 수 있지만, 매번 새로운 연결을 맺어야 하는 비용이 발생

무상태성
    서버가 클라이언트의 이전 상태를 기억하지 않는 특징
    각 요청은 독립적이며 서버는 요청에 필요한 모든 정보를 매번 전달받아야함
    이 특징 덕에 서버 확장 용이

### HTTP 메세지 구조
요청 메세지
    시작 라인: 메소드, 타겟 경로, HTTP 버전이 포함
    헤더: 요청에 대한 부가 정보가 들어감
    바디: 서버로 보낼 실제 데이터가 담기는 부분

응답 메세지
    시작 라인: HTTP 버전, 상태 코드, 상태 문구가 포함
    헤더: 응답에 대한 부가 정보가 들어감
    바디: 서버가 클라이언트에 전달하는 실제 데이터가 담김

### HTTP 메소드
클라이언트가 서버에 요청의 목적을 알리는 방법

메소드  /   설명
GET    /   리소스의 조회를 의미
POST   /   요청 데이터 처리이며 주로 신규 리소스 등록에 사용
PUT    /   리소스를 완전히 대체하거나 생성
PATCH  /   리소스의 일부만 수정
DELETE /   리소스의 삭제를 의미

### HTTP 상태 코드
서버가 클라이언트의 요청이 성공했는지 또는 실패했는지 알려주는 3자리 숫자

2xx - 성공
    200: 요청이 정상적으로 처리된 상태
    201: 요청이 성공하여 새로운 리소스가 생성된 상태

3xx - 리다이렉션
    301: 리소스의 위치가 영구적으로 이동된 상태
    302: 리소스의 위치가 일시적으로 이동된 상태

4xx - 클라이언트 오류
    400: 잘못된 요청 문법 등으로 서버가 처리할 수 없는 상태
    401: 인증되지 않은 사용자임을 나타냄
    403: 권한이 없어 접근이 거부된 상태
    404: 요청한 리소스를 찾을 수 없는 상태

5xx - 서버 오류
    500: 서버 내부의 문제로 오류가 발생한 상태
    503: 서버 과부하 등으로 일시적으로 서비스를 이용할 수 없는 상태

### HTTP와 HTTPS
HTTPS는 HTTP에 보안 계층을 추가한 프로토콜
    암호화: 데이터를 암호화하여 전송하므로 중간에 데이터를 가로채더라도 내용을 알 수 없음
    데이터 무결성: 전송 중에 데이터가 변조되지 않았음을 보장
    인증: 연결하려는 서버가 믿을 수 있는 서버인지 확인
