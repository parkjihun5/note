### 데이터 모델
데이터 모델은 데이터가 어떻게 구성되어야 하는지를 정의한 데이터 설계도  
어떤 정보가 포함되어야 하고 각 정보가 어떤 타입(숫자, 문자 등)이어야 하는지 약속하여 데이터의 형식을 통일하는 역할을 함  

### Pydantic
Pydantic은 파이썬의 타입 힌트를 이용하여 데이터를 검증하고 설정 관리를 수행하는 라이브러리  
실행 시점에 데이터 형식을 강제하며, 정의된 모델과 일치하지 않는 데이터가 입력될 경우 정해진 규칙에 따라 오류를 발생시킴  
- 데이터 검증(validation)  
들어온 값이 숫자인지, 문자열인지, 필수 값이 빠지지 않았는지 자동으로 체크  
- 직렬화(Serialization)
파이썬 객체를 JSON 형태의 텍스트로, 또는 그 반대로 아주 쉽게 변환  

### BaseModel
- BaseModel은 Pydantic에서 데이터 모델을 생성하기 위해 상속받아야하는 클래스  
- 이 클래스를 상속받아 정의된 클래스는 각 필드의 데이터 형식을 선언하며, 입력된 데이터를 해당 형식에 맞게 변환하거나 검증하는 기능을 가지게 됨  

### 데이터 검증 및 타입 강제 메커니즘
Pydantic은 인스턴스 생성 시점에 각 필드에 할당된 데이터 형식을 확인함  
선언된 형식과 호환됮 않는 데이터가 입력되면 검증 어류(ValidationError)를 발생시켜 프로그램의 안정성을 확보  

### 기본값 설정 및 선택적 필드
필드 정의 시 등호를 사용하여 기본값을 지정할 수 있음  
기본값이 설정된 필드는 데이터 입력 단계에서 생략이 가능하며, 파이썬의 | 연산자를 통해 데이터가 없는 상태(None)도 허용할 수 있음  

### 중첩된 모델 구조
하나의 데이터 모델 내부에 다른 BaseModel 클래스를 필드 형식으로 정의할 수 있음  
이는 복잡한 계층 구조를 가진 데이터를 구조화하고 하위 객체까지 일괄적으로 검증하는데 사용  
모델들의 리스트를 필드로 가질 수 있음  

### 모델의 직렬화
검증이 완료된 객체를 파이썬 딕셔너리나 JSON 문자열로 변환하는 기능을 제공  
이는 API 응답을 생성하거나 데이터를 저장할 때 활용
