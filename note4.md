## 자료구조 기초

프로그램을 작성하다보면 같은 자료 형태의 데이터를 여러개를 작성해야하는 경우가 있음
하나하나 변수를 할당하면 너무 많음
그래서 배열이 사용됨
배열과 함께 반복문이 많이 사용됨

배열(Array) = 같은 자료형의 데이터를 연속적인 메모리 광간에 저장할 수 있는 자료구조
특징 :
    같은 자료형만 연속으로 저장할 수 있음
    선언한 배열의 내용물은 변경이 가능
    선언한 배열의 길이는 변경할 수 없음
    메모리와 같은 하드웨어와 직접 관련이 있기 떄문에, 파이썬과 같이 편의성을 우선으로 하는 언어는 제공을 하지 않는 경우가 있음

배열의 선언과 초기화
    배열의 크기를 지정하면서 일반 변수와 유사하게 선언
    배열 각 원소는 0, False, Null로 자동으로 초기화 됨
    
인덱스
    배열의 각 원소는 0부터 시작하는 인덱스를 통해 접근
    인덱스로 접근한 원소의 값을 수정하거나 가져올 수 있음
    선언한 크기를 넘는 인덱스에 접근하려고 하면 오류 발생

배열의 순회 조회법(Iteration)
    배열을 순회하는 경우, for문과 인덱스를 이용할 수 있음
    배열을 순회하는 다른 방법으로, 향상된 for 루프(인핸스드 for 루프)를 이용할 수도 있음

다차원 배열 : 여러개의 인덱스로 2차원 이상을 표현할 수 있는 배열
    다차원을 표현할 수 있지만, 실제로는 메모리에 선형적으로 할당함
용도 :
    수학적으로 행렬(Mateix)을 표현하기 좋음
    선형대수학의 행렬 연산에 사용
    각종 수학 연산, 컴퓨터 그래픽, 머신러닝, 인공지능 등에 필수

가까이에서 배열을 찾을 수 있는 위치
컴퓨터 이미지
    화소(픽셀, Pixel): 컴퓨터 화면의 화상을 구성하는 최소 단위
    화소의 좌표: 각 화소의 위치는 2차원 배열의 인덱스로 나타낼 수 있음
    화소의 색: 각 화소의 색은 세 숫자의 조합으로 나타내는데, 2차원 배열 원소의 값으로 나타낼 수 있음

IT 개발 현장에서의 쓰임새?
다양한 자료구조를 구현하는 기초로 사용
    문자열, 리스트, 큐, 스택 등, 여러 원소를 저장하는 특징이 있는 모든 자료구조에서 배열을 사용
고성능의 수학연산이 필요한 곳
    컴퓨터 그래픽스, 머신러닝, 과학 시뮬레이션 등

심화 주제 :
    리스트 - 배열 다음으로 기초적인 자료구조, 배열처럼 데이터를 선형으로 저장하는 자료구조
    문자열 연산 - 문자열이 배열을 기초로 함, 고급 프로그래밍 언어일 수록 배열을 직접 사용해서 문자열을 다루는 일이 많지는 않으나 성능을 중요시 여기는 프로그램을 작성할 때 문자열 연산과 배열간의 관계를 잘 알아야 함
    선형대수 - 조금 더 공학적이고 과학적인걸 알고 싶으면 선형대수는 꼭 짚고 넘어가야 함
    컴퓨터 그래픽스
